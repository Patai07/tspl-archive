# TSPL Project Log - 2026-05-17

## 🛠️ รายการแก้ไขและอัปเดตระบบ (TSPL Control Center)

วันนี้ได้ทำการ Stabilize ระบบการจัดการทรัพย์สิน (Asset Management) และระบบตรวจสอบความซ้ำซ้อนระหว่าง Google Drive และฐานข้อมูล โดยมีรายละเอียดดังนี้:

### 1. ระบบเปรียบเทียบชื่อขั้นสูง (Super Ultra Clean Matching)
- **ปัญหา:** ชื่อโฟลเดอร์ใน Drive และชื่อใน Google Sheet มักไม่ตรงกัน (มีเลขนำหน้า, มีวงเล็บภาษาอังกฤษ, มีคำว่า 'ลาย' หรือ 'ลวดลาย') ทำให้ Dashboard แสดงยอด "Unscanned" ผิดพลาด
- **การแก้ไข:** 
    - พัฒนาฟังก์ชัน `super_clean` และ `_norm` ให้ทำความสะอาดชื่อก่อนเปรียบเทียบ (ตัดตัวเลข, ตัดนามสกุลไฟล์, ตัดคำนำหน้า 'ลาย/ลวดลาย', ล้างอักขระพิเศษ)
    - เพิ่มระบบ **Unicode Normalization (NFC)** เพื่อแก้ปัญหาตัวอักษรไทยจาก Mac และ Windows/Web ที่มีรหัสไม่ตรงกัน

### 2. ระบบ Manual Mapping สำหรับกรณีพิเศษ
- **ปัญหา:** บางลายมีชื่อที่แตกต่างกันอย่างสิ้นเชิง เช่น โฟลเดอร์ `ลวดลายดอกไม้ประดับอัญมณี` แต่ในฐานข้อมูลคือ `สังวาลพระพุทธชินราช`
- **การแก้ไข:** เพิ่มตาราง `manual_map` ในโค้ดทั้ง Dashboard และตัวแสกน Haiku เพื่อล็อคเป้าจับคู่ชื่อเหล่านี้โดยเฉพาะ ทำให้ระบบจำได้แม่นยำ 100%

### 3. ป้องกันรหัส Symbol ID ซ้ำซ้อน (Global ID Check)
- **ปัญหา:** เดิมระบบเช็ค ID สูงสุดแค่ใน Tab ปัจจุบัน ทำให้เวลาแสกนใหม่มีโอกาสออกเลข ID ซ้ำกับของเก่า (เช่น TSP-LST-FAU-001 ซ้ำกันสองลาย)
- **การแก้ไข:** ปรับปรุงให้ตัวแสกน Haiku เข้าไปกวาดรหัสจาก **ทุก Tab** (Sheet 1 และ Haiku_Scan_*) มาตรวจสอบก่อนออกรหัสใหม่เสมอ

### 4. การปรับปรุง UI/UX และเสถียรภาพ
- เพิ่มปุ่มทางลัด **DRIVE** ในแถบ Sidebar เพื่อให้เข้าถึง Master Folder ได้ทันที
- แก้ไขข้อผิดพลาด Syntax Error และปัญหา Server ล่มจากการประมวลผลซ้ำซ้อน
- ปรับปรุง Dashboard ให้ตรวจสอบยอดงานค้างจากทุก Tab สแกน
